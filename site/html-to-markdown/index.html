How to convert HTML to Markdown

  1. Paste your HTML into the input box on the left — the output updates live as you type.
  2. Pick a bullet marker, code block style, and heading style from the options below the editor to match your target Markdown flavor.
  3. Switch the output panel between Raw (the literal Markdown source) and Rendered (a live preview re-parsed from that Markdown) with the toggle above it.
  4. Copy the Markdown with Copy Markdown, or use Download .md to save it as a file.

What this converter actually does

The conversion is handled by turndown, the standard HTML-to-Markdown library, with the turndown-plugin-gfm extension turned on so GitHub-Flavored constructs — tables, strikethrough (<del>/<s>), and task-list checkboxes — convert to their Markdown equivalents instead of degrading to plain text. Headings, lists (ordered and nested), links, images, blockquotes, inline code, and fenced code blocks with a detected language all round-trip cleanly. Switch to Rendered to sanity-check the result: the generated Markdown is re-parsed with marked and sanitized with DOMPurify before it's shown, so what you see is exactly what a Markdown renderer downstream — GitHub, a static site generator, an LLM prompt — would produce from the text you're about to copy.

Presentation-only markup has no Markdown equivalent and is intentionally dropped: inline style attributes, custom CSS classes, colors, fonts, and multi-column or grid layouts all disappear, leaving just the structural content. That's normally what you want — Markdown is a content format, not a layout format — but it does mean a visually elaborate page will convert into much plainer Markdown than it looked like in the browser.

Common edge cases

Nested tables, <iframe>s, forms, and inline SVG have no Markdown representation and are dropped or left as raw HTML passthrough, depending on the tag. Table cells containing line breaks or pipe characters (|) are escaped so the table doesn't break — if a cell's content is complex enough, consider whether a table is the right output at all. If you're pasting content copied from a live webpage rather than its HTML source, make sure what landed in the box is actual markup and not rendered plain text — browsers usually require "Copy" from dev tools' Elements/Inspect panel, not a normal page selection, to get real HTML onto the clipboard. Deeply nested <div> wrappers (common in exports from rich text editors and CMSs) convert fine but can produce more blank lines than a hand-written document would — that's the converter faithfully reflecting the source's block structure, not a bug.

Frequently asked questions

Is this HTML to Markdown converter free?

Yes. It's free to use, with no signup, and there's no limit on how much HTML you can convert.

Does my HTML get uploaded anywhere?

No. The conversion happens entirely in your browser using JavaScript — your HTML is never sent to a server.

Does it convert tables, task lists, and strikethrough?

Yes. Tables, task lists (checkbox items), and strikethrough all convert to their GitHub-Flavored Markdown equivalents, via the turndown-plugin-gfm extension.

Can I paste HTML copied straight from a webpage or Word?

Yes — paste the HTML source directly into the input box. If you're copying rendered content from a browser or Word rather than viewing source, use your browser's "Inspect" or "View Source" to grab the actual markup first, since a plain clipboard paste normally lands as rendered text, not HTML.

Why does the Markdown output look different from the original HTML?

Markdown can't represent everything HTML can — things like inline styles, colors, custom classes, and multi-column layouts have no Markdown equivalent and are dropped. The converter keeps structure (headings, lists, links, tables, code, emphasis) and discards presentation.

Can I choose the bullet character or code block style?

Yes — the options below the editor let you pick the bullet marker (-, *, or +) and switch code blocks between fenced (```) and indented style.

Can I preview the Markdown instead of reading raw syntax?

Yes — use the Raw / Rendered switch above the output box. Rendered re-parses the generated Markdown and shows a sanitized live preview; Raw shows the literal Markdown source.
