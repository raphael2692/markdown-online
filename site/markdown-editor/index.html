How to use the Markdown editor

  1. Start typing in the Write pane — the Preview pane updates as you go, so you always see exactly how the final Markdown will render.
  2. Use the toolbar, or shortcuts like Ctrl/Cmd+B for bold, Ctrl/Cmd+I for italic, and Ctrl/Cmd+K for a link, to insert formatting without memorizing syntax.
  3. Switch to Write-only or Preview-only with the toggle above the toolbar when you want the full width for typing or for reviewing the rendered result.
  4. When you're done, copy the Markdown, copy a rich version for Word or Google Docs, download it as a .md or .html file, or print it straight to PDF.

A live preview without losing the syntax

This editor pairs a plain-text Markdown pane with a preview rendered by marked, the same GitHub-Flavored Markdown parser used across this site, so tables, task lists, strikethrough, and autolinked URLs all render correctly by default — no configuration needed. The two panes scroll together in Split view, so a heading near the top of a long draft stays lined up with its rendered counterpart instead of drifting apart as you write. The toolbar exists for anyone who doesn't want to remember that a link is [text](url) or that a quote needs a > at the start of the line — select some text, click a button, and the syntax is inserted around it (or a sensible placeholder, if nothing is selected). The preview is always sanitized with DOMPurify before it touches the page, so pasted content with a stray <script> tag renders inert rather than executing.

Three optional toggles above the toolbar extend the preview. Smart typography turns straight quotes into curly quotes and --/--- into en/em dashes. Render math loads KaTeX so $inline$ and $$block$$ LaTeX renders as typeset equations. Render diagrams loads Mermaid so fenced ```mermaid blocks render as flowcharts and sequence diagrams. All three are off by default, apply to the preview, the rich copy, the HTML download, and Print / Save as PDF alike, and only download their (larger) supporting libraries once switched on.

Your draft is saved locally — nothing is uploaded

About half a second after you stop typing, the editor saves your current text to your browser's local storage and picks it back up automatically the next time you open this page in the same browser — so an accidental tab close or refresh doesn't cost you a paragraph. That save never leaves your machine: there's no account, no server copy, and no sync across devices, which is a real trade-off against cloud editors — if you need the same draft on your phone and your laptop, this isn't that tool. What you get in exchange is a genuine privacy guarantee for anything you'd rather not send to a third party, like an unpublished draft or notes pasted from an internal doc. Hit Clear at any time to wipe both the editor and the saved copy; otherwise it just sits in that one browser until you overwrite it by writing something new.

Formatting quirks worth knowing

Lists need a blank line before and after them to render as a proper list rather than getting absorbed into the surrounding paragraph, and a nested item needs at least two spaces of indent under its parent. A literal asterisk or underscore — say, describing a variable named *args — needs a backslash in front of it (\*args) or it gets read as emphasis. Fenced code blocks with a language tag after the triple backticks (```js) get a language-* class in the output, but this editor's preview doesn't apply syntax-highlighting colors to it — that's a rendering choice for wherever the Markdown ends up, not something baked into this tool.

Frequently asked questions

Is this Markdown editor free to use?

Yes. It's free, with no signup and no limit on how much you write.

Does my writing get uploaded to a server?

No. The editor and preview run entirely in your browser using JavaScript. The only place your draft is stored is your browser's own local storage — it never touches a server.

Will I lose my draft if I close the tab or refresh the page?

No. Your draft autosaves to local storage about half a second after you stop typing, and reloads automatically the next time you open this page in the same browser. Use the Clear button to remove it explicitly.

Can I export what I write as Word, PDF, or HTML?

Yes. Use "Copy for Word / Docs" to paste formatted content straight into a document, "Download HTML" for a standalone file, or "Print / Save as PDF" to generate a PDF through your browser's print dialog.

What keyboard shortcuts does the editor support?

Ctrl/Cmd+B for bold, Ctrl/Cmd+I for italic, and Ctrl/Cmd+K for a link, alongside toolbar buttons for headings, lists, quotes, and code.

Can I write math equations or diagrams?

Yes. Turn on "Render math" for KaTeX-powered $inline$ and $$block$$ LaTeX, or "Render diagrams" for Mermaid-powered ```mermaid flowcharts and sequence diagrams — both apply to the preview, rich copy, HTML download, and Print / Save as PDF.

What does "Smart typography" do?

It converts straight quotes into curly quotes and --/--- into en/em dashes — useful for polished writing, off by default so the raw Markdown you typed stays exactly as-is.
