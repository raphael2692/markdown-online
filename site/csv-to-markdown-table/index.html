How to convert CSV to a Markdown table

  1. Paste your CSV into the input box — or click JSON to paste an array of objects instead.
  2. Toggle First row is header if your CSV's first line is data rather than column names, and pick left, center, or right Alignment.
  3. Switch the output panel between Raw (the literal Markdown source) and Rendered (a live preview) to sanity-check the result.
  4. Use Copy Markdown for a README or docs page, Download .md to save a file, or Copy for Word/Docs to paste a real formatted table into Word or Google Docs.

What this converter actually does

CSV parsing is handled by PapaParse, which correctly handles quoted fields — so a value like "Smith, John" stays one cell instead of splitting on its internal comma. JSON input is treated as an array of objects: the converter takes the union of every key across all objects as the column list, so the table still lines up even if some rows are missing a field — those cells are just left blank. Every column is padded to a fixed width rather than left ragged, which is the detail most quick table generators skip, and the alignment dropdown writes the standard :--- / :---: / ---: colon syntax into the separator row so renderers that respect Markdown table alignment (GitHub, most static site generators) show the columns aligned the way you picked.

CSV quirks and edge cases

A pipe character (|) inside a cell would normally break a Markdown table's column boundaries, so it's automatically escaped to \|; line breaks inside a quoted CSV field are converted to <br>, since Markdown tables can't contain literal newlines within a row. Rows with fewer columns than the header are padded with empty cells rather than shifting the rest of the row out of alignment. If your source is a spreadsheet export, watch for a leading byte-order-mark (BOM) or a semicolon delimiter from a European-locale export — PapaParse detects standard comma and tab delimiters automatically, but an unusual delimiter can cause the whole file to parse as a single column, which shows up immediately as a one-column table in the preview.

Frequently asked questions

Is this CSV to Markdown table converter free?

Yes. It's free to use, with no signup, and there's no limit on how many rows you can convert.

Does my CSV or JSON data get uploaded anywhere?

No. Parsing and table generation happen entirely in your browser using JavaScript — your data is never sent to a server.

Can I convert JSON as well as CSV?

Yes — switch to the JSON tab and paste an array of objects. The union of all keys across the array becomes the table's columns, so it still works if some objects are missing a field.

What happens to commas, quotes, or line breaks inside a cell?

Quoted CSV fields (values wrapped in double quotes, including ones containing commas) are parsed correctly rather than split apart. Inside the generated table, any pipe character is escaped to \| and line breaks are converted to <br> so they can't break the table structure.

Can I control column alignment or skip the header row?

Yes — the alignment dropdown sets left, center, or right alignment for every column using standard Markdown colon syntax, and the "First row is header" checkbox lets you tell the converter whether your CSV's first line is column names or the first data row.
