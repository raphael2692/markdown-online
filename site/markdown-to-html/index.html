Markdown to HTML Converter

Paste Markdown, get clean HTML instantly. Everything runs locally in your browser — your text is never uploaded to a server.

How to convert Markdown to HTML

  1. Paste your Markdown into the input box on the left — the output updates live as you type.
  2. Turn on "Line breaks as <br>" if you want single newlines to become line breaks (off by default, matching GitHub).
  3. Switch the output panel between Raw (the literal HTML source) and Rendered (a live preview) with the toggle above it.
  4. Copy the HTML with Copy HTML, or use Copy for Word / Docs to paste formatted content straight into a document, or Download HTML to save it as a file.

What this converter actually does

The conversion is handled by marked, a Markdown parser that follows the CommonMark spec with GitHub-Flavored Markdown extensions turned on by default — so tables, task lists, strikethrough, and autolinked URLs all work without any configuration. In Raw view, the output box shows the literal HTML marked generates: headings as <h1>–<h6>, fenced code blocks as <pre><code> with a language-* class, tables as real <table> markup — left exactly as generated (not sanitized), since the point of this tool is to hand you your HTML verbatim so you can inspect or embed it. Switch to Rendered to see how that markup actually looks, run through DOMPurify first so anything unsafe in your Markdown — a stray <script> tag, an onerror handler — is stripped before it touches the page. "Copy for Word / Docs" always uses that same sanitized rendering, regardless of which view is showing, since that copy is meant to be pasted straight into a document rather than read as source.

Common edge cases

A single newline between two lines of text is treated as a plain space, not a line break — that's standard Markdown behavior and matches how GitHub renders READMEs. If you're converting notes written with the "every line is a new line" assumption (common when pasting from chat apps), turn on the line-breaks option above. Nested lists follow indentation: a sub-item needs at least two spaces of indent under its parent to nest correctly, and mixing tabs and spaces can throw off list nesting in ways that are hard to spot by eye. Fenced code blocks are output as plain text with a language class attached — this tool doesn't apply syntax-highlighting colors to the output HTML, since that's a rendering concern for wherever you embed the HTML, not something to bake into the markup itself.

Frequently asked questions

Is this Markdown to HTML converter free?

Yes. It's free to use, with no signup, and there's no limit on how much Markdown you can convert.

Does my Markdown get uploaded anywhere?

No. The conversion happens entirely in your browser using JavaScript — your Markdown is never sent to a server.

Does it support GitHub-Flavored Markdown, like tables and task lists?

Yes. Tables, task lists, strikethrough, and autolinks are all supported and enabled by default.

Can I get a complete HTML document instead of just a fragment?

Yes — toggle "Full HTML document" to wrap the output in a complete page with basic styling, ready to save as a standalone file.

Can I see a rendered preview instead of the raw HTML?

Yes — use the Raw / Rendered switch above the output box. Rendered shows a live, sanitized preview of the markup; Raw shows the literal HTML source.

Why might my HTML output differ from GitHub's Markdown renderer?

This tool uses the marked library, which follows the CommonMark and GFM spec closely but isn't byte-for-byte identical to GitHub's own renderer — differences mostly show up in edge cases like nested list indentation or raw HTML passthrough.
